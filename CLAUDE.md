# Tests

After every code change, check whether existing tests need updating and whether new tests should be added. Cover new behaviour; remove or rewrite tests that assert behaviour that no longer exists.

# Commit messages

When asked for a commit message, provide only the message text — no shell command wrapping. Format:

```
<short imperative summary>

<one or two sentences on why, not what>

- bullet list of notable changes
```

Make the commit message copyable
